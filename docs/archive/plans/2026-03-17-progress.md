**Critical Bug Fixed:**
The initial implementation of `buildHopsFromSignature()` only populated the `Host` field for jump hosts, which would cause connection establishment to fail. The fix:

1. **Before (WRONG):**
```go
hop := &tunnel.HopConfig{
    Host: jumpHost,
    // HostName, Port, User, etc. will be resolved by SSH config parser
}
```

2. **After (CORRECT):**
```go
func buildHopFromHost(parser *config.Parser, host string) (*tunnel.HopConfig, error) {
    hop := &tunnel.HopConfig{Host: host}

    if parser != nil {
        hostConfig, err := parser.GetHostConfig(host)
        if err == nil {
            hop.HostName = hostConfig.HostName
            hop.Port = hostConfig.Port
            hop.User = hostConfig.User
            hop.IdentityFile = hostConfig.IdentityFile
            hop.CertificateFile = hostConfig.CertificateFile
        }
    }

    // Apply defaults
    if hop.HostName == "" {
        hop.HostName = host
    }
    if hop.Port == 0 {
        hop.Port = 22
    }

    return hop, nil
}
```

**Why This Matters:**
- Connection pool's `dialSSH()` calls `Establish(hops, builder)`
- SSH config resolution needs `HostName`, `Port`, `User` fields
- Without these, connection creation would fail
- The fix ensures all required SSH config fields are populated
